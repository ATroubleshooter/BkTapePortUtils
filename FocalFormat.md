# Изученный опытным путем формат BIN-файла для программы на Фокале

Создание BIN-файла из исходника на Фокале позволяет загрузить программу в
эмуляторе GID обычной командой `L G NAME`

* Первые 26 байт - заголовок
* Первые два байта заголовка - $EA и $03
* Следующие два байта заголовка - общий размер файла в LittleEndian
* Оставшиемся 22 байта заголовка - последовательность 
$14, $0, $0, $0, $43, $27, $20, $20, $E6, $EF, $EB, $E1,
$EC, $2D, $E2, $EB, $30, $30, $31, $30, $8E, $00
* Далее идут строки программы, заканчивающие на символ $8E и при необходимости,
дополняемые нулем до четного размера.
* В строках символы пробела заменяются на код $80, также заменяются символы
`+-/*^()[],;=` на соответствующие коды (список кодов в конце документа)
* Перед каждой строкой - идут четыре байта заголовка строки.
* Для всех строк, кроме последней - первые два байта заголовка строки это
размер строки плюс 2.
* Для последней строки - это магическая константа $FBFE, от которой вычли
длины всех строк (кроме последней), увеличенные на 4.
* Как для первой, так и для последней строк - последние два байта заголовка
строки это закодированный номер строки: значение строки до точки кодируется
просто как байт и ставится в байт 3, а значение строки после точки вычисляется
по формуле `Trunc(256*(NUM/100))` и ставится в байт 2.

## Коды замены байт в строках Фокала

* пробел -> $80
* `+` -> $81
* `-` -> $82
* `/` -> $83
* `*` -> $84
* `^` -> $85
* `(` -> $86
* `[` -> $87
* `)` -> $89
* `]` -> $8A
* `,` -> $8C
* `;` -> $8D
* `=` -> $8F

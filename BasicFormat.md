# Изученный опытным путем формат серии ASC-файлов для программы на Бейсике

Серии ASC-файлов позволяют загрузить программу на Бейсике в эмулятор GID,
обычной командой `LOAD "NAME"`

Формат серии файлов:
* В конец файла программы добавляется байт $1A
* Программа разбивается на блоки по 256 байт
* Блоки помещаются в файлы с именами `NAME.ASC #ddd.BIN`, где NAME - 
имя программы, дополненное пробелами справа до 6 символов, а ddd - номер блока,
начиная с 0, дополненный нулями слева до трех цифр. Например, третий
файл программы `GAME` будет называться `GAME  .ASC #002.BIN`
* Каждый файл начинается с заголовка на четыре байта. Первые два байта -
"магические значение" $EE и $3D. Следующие два байта - размер блока данных,
закодированный в LittleEndian.
* Также добавляется один файл для любой серии файлов, с именем 
`NAME.ASC.BIN`, где NAME - имя программы, дополненное пробелами справа до 6 символов.
Его содержимое - "магическая последовательность" $EE, $3D, $2, $0, $0, $0,
что соответствует программе из двух нулевых символов.

## Порядок создания WAV-файла из серии ASC.BIN файлов

Все файлы, включая последний с именем без номера, записываются в один WAV-файл,
где каждый из них предваряется соответствующим именем на ленте.
* Для файлов с номерами - название фрагмента на ленте это `NAME.ASC #ddd\26`
где NAME - имя программы, дополненное пробелами справа до 6 символов, ddd - номер блока,
начиная с 0, дополненный нулями слева до трех цифр, а \26 - символ с кодом 26 (десятичная система).
* Для файла без номера (он идет последним в WAV-файле), название на ленте иное:
`NAME.ASC\32\32\0\0\0\0`
где NAME - имя программы, дополненное пробелами справа до 6 символов,
\32 - пробелы, а \0 - символы с кодом 0.
